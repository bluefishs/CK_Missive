## 四、NIM 狀態說明

NIM **未消失**，而是被 vLLM 暫時替代：

| 項目 | NIM | vLLM |
|------|-----|------|
| 映像 | ✅ 保留 (26.8+30.9GB) | ✅ 啟用 (30.2GB) |
| 後端 | TRT-LLM (需 engine build) | vLLM (直接載入) |
| RTX 4060 8GB | ❌ VRAM 不足 build engine | ✅ AWQ 量化 5.2GB |
| Profile 選擇 | ❌ 不支援 (1.8.3) | ✅ --model 指定 |
| API | OpenAI-compatible | OpenAI-compatible |
| 回歸條件 | NIM 新版支援 vLLM profile | -- |

**回歸路徑**: 當 NVIDIA 發布支援 `--profile vllm` 的 NIM 版本時，只需修改 docker-compose 映像名稱即可切回。
