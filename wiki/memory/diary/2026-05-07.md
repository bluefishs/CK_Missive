### Lesson 候選（強候選為 LESSON_REGISTRY）

**L23 候選 — Half-Wired Anti-Pattern Stacking**：
單一現象（用戶看到不該看的選單）背後可能是 **多層獨立 bug 疊加**。修第一層後外觀沒變
不代表沒修，而是被第二層遮住。Debug 邏輯：每修完一層必須**穿透下一層繼續驗證**，
不能憑「測一次仍有問題就回滾」誤判。

**L24 候選 — Dev Mode Override Trap**：
`VITE_AUTH_DISABLED=true` 為求方便但 **強制覆蓋真實用戶身份** → 內網開發者永遠看到
superuser 視角，無法測試其他 role。修法：dev mock 只在「沒有真實 user_info」時介入，
有真用戶就尊重其身份。所有 frontend dev override 都該遵循此「opt-in fallback」原則。

**L25 候選 — JSON-as-TEXT Schema Drift**：
DB 用 TEXT 存 JSON 是常見折衷（ORM 不全支援 JSONB），但 endpoint 返回前必須 parse。
未 parse 直送前端時，`'[]'.length=2` 等假陽性會以詭異方式破壞 client-side filter。
所有 `Column(Text, comment="JSON")` 在 dict 化前都要過 helper parse + 加 unit test 鎖定。
